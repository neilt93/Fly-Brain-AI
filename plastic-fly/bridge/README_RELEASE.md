# Brain-Body Bridge

A sensorimotor interface that connects a whole-brain connectome model
(139,255 FlyWire neurons simulated as LIF units in Brian2) to a
biomechanical fly body (FlyGym/MuJoCo). The bridge translates body
state into sensory neuron firing rates, steps the brain, reads
descending neuron activity, and converts it into locomotion commands
that modulate a CPG-based motor layer. The brain never touches raw
joint torques -- it drives behavior through the same descending neuron
pathway used by real flies.

## Architecture

```
 FlyGym body (MuJoCo)
       |
       v
 FlyGymAdapter          -- extract BodyObservation from FlyGym obs dict
       |
       v
 SensoryEncoder         -- body state -> Poisson rates for sensory neurons
       |
       v
 BrainRunner            -- Brian2 LIF network (139k neurons, 15M connections)
       |
       v
 DescendingDecoder      -- DN firing rates -> {forward, turn, rhythm, stance}
       |
       v
 VNCLite (optional)     -- bilateral premotor dynamics with body feedback
       |
       v
 LocomotionBridge       -- LocomotionCommand -> CPG-modulated joint angles
       |
       v
 FlyGym body (MuJoCo)
```

## Prerequisites

- Python 3.10+
- flygym 1.2.1 (includes MuJoCo)
- brian2
- numpy (tested with 2.0.2)
- pandas (for loading connectome parquet)
- torch, scipy, matplotlib (for experiments)

## Quickstart

```bash
# 1. Generate neuron population files (sensory IDs, readout IDs, decoder groups)
cd plastic-fly
python scripts/select_populations.py

# 2. Test the loop with a fake brain (no Brian2, no connectome data needed)
python experiments/closed_loop_walk.py --fake-brain --body-steps 2000

# 3. Run with the real 139k-neuron brain (requires brain-model repo)
python experiments/closed_loop_walk.py --body-steps 5000
```

## Modules

| Module | Description |
|---|---|
| `interfaces.py` | Shared dataclasses: BodyObservation, BrainInput, BrainOutput, LocomotionCommand. Every other module imports from here. |
| `config.py` | BridgeConfig: paths, timing constants, encoding parameters. Single source of truth for all tunable bridge settings. |
| `sensory_encoder.py` | Channel-aware encoder mapping body state to Poisson rates. Supports gustatory, proprioceptive, mechanosensory, vestibular, olfactory, visual, and LPLC2 looming channels. |
| `brain_runner.py` | Brian2BrainRunner (real 139k LIF network with PoissonGroup input) and FakeBrainRunner (echo stub for testing). Factory function `create_brain_runner()` selects between them. |
| `descending_decoder.py` | Partitions readout DNs into five functional groups (forward, turn_left, turn_right, rhythm, stance) and maps mean firing rates to LocomotionCommand via tanh nonlinearities. |
| `locomotion_bridge.py` | CPG + PreprogrammedSteps motor layer. Takes LocomotionCommand and produces joint angles and adhesion signals for FlyGym. |
| `flygym_adapter.py` | Extracts BodyObservation from FlyGym's raw observation dictionary. Handles joints, contact forces, vision, and olfaction. |
| `vnc_lite.py` | Optional bilateral premotor dynamics layer (VNCLite). Adds temporal smoothing, persistence, mutual inhibition, and body feedback between the decoder and the locomotion bridge. |

## Data Files

Generated by `scripts/select_populations.py` into `data/`:

| File | Contents |
|---|---|
| `sensory_ids.npy` | FlyWire root IDs for sensory input neurons |
| `readout_ids.npy` | FlyWire root IDs for descending readout neurons |
| `decoder_groups.json` | DN-to-group assignment (forward, turn_left, turn_right, rhythm, stance) |
| `channel_map.json` | Sensory neuron-to-channel assignment (gustatory, proprioceptive, etc.) |

## External Dependencies

The brain model requires connectome data from the
[Drosophila_brain_model](https://github.com/philshiu/Drosophila_brain_model)
repository (not included). Clone it as a sibling directory:

```
parent/
  brain-model/          # philshiu/Drosophila_brain_model
    Completeness_783.csv
    Connectivity_783.parquet
    sez_neurons.pickle
  plastic-fly/
    bridge/
    ...
```

BridgeConfig expects `brain-model/` at `../../brain-model` relative to
the bridge package. Override with `BridgeConfig(brain_repo_root=...)`.
